RAGAS Evaluation Flowchart - Layout Options

Select the visual layout structure you prefer for your academic report.

A

Vertical Flow (Strictly Sequential)

A single linear vertical sequence. From PDF/JSONL inputs, to PDF extraction & chunking, hybrid retrieval, LLM response generation, and finally RAGAS metric calculations. Clean, easy to follow, and fits perfectly in a standard portrait A4 column.

B

Two-Stage Functional Blocks

Splits the diagram into two distinct, stacked phases: (1) RAG System Pipeline (the application under test) and (2) RAGAS Evaluation Engine (the judge pipeline). Clearly separates the system logic from the evaluation logic.

C

Iterative CRISP-eval Loop

A sequential layout but with a feedback loop arrow running from the final evaluation scores (CSV report) back to the system hyperparameters (chunking overlap, Fusion Method, local LLM selection), emphasizing optimization and continuous integration.

Recommended: Option A (Vertical Flow) or Option B (Two-Stage)

  • Option A is very simple and matches standard academic system architecture diagrams.
  • Option B highlights that RAGAS is an external evaluation harness querying the RAG pipeline.

Considerations

  • Option C adds complexity that may not be necessary if you only run evaluation on-demand.
