RAGAS Evaluation Flowchart Design (Vertical Sequence)

A portrait-oriented, step-by-step flowchart from inputs to the final RAGAS metrics CSV. Here is the visual layout:

1. Input Sources
PDF Lecture Notes (e.g., in media/data_source/) OR pre-defined evaluation dataset in JSONL format
2. Text Preparation & Smart Chunking
Extract text using PDFLoader → Split into 500-character chunks with 100-character overlap
3. QA Pair Generation (LLM-based)
Prompt generator LLM (Gemini/Qwen) to output JSON array of {question, ground_truth} pairs → Save as dataset JSONL
4. Hybrid Retrieval Engine
For each question: query Dense Retriever (FAISS) + Sparse Retriever (BM25) → Fuse using Reciprocal Rank Fusion (RRF, k=60) → Select top top_k context chunks
5. Answer Generation
Format retrieved context + question → Send to LLM generator (local Llama.cpp / Gemini) → Produce answer
6. Dataset Assembly
Compile evaluation records containing: question, contexts, answer, and ground_truth into Hugging Face Dataset format
7. RAGAS Evaluation Engine
Initialize ChatOpenAI (Judge LLM wrapper) + _LocalRagasEmbeddings (MiniLM embeddings). Run RAGAS evaluate() to calculate metrics:
• Faithfulness
• Answer Relevancy
• Context Precision
• Context Recall
8. Evaluation Output Reports
Write complete dataframe with metrics to CSV report + Log aggregate metrics summary (overall score report)
